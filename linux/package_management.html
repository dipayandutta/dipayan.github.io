    yum vs dnf and also apt-get

    Author:Dipayan Dutta | 06-12-2016

    Here i am discussing the package managment command for two most widely used and popular linux distro's One is Redhat and other is Debian , althoug Arch which supports pacman will be also discussed later , other distro's supports other package managment tools But yum and apt-get this two are most common in linux package managment field.yum Yellowdog updated Modified , it is the package manager for the rpm based distribution such as Redhat , fedora , centos , yum was also a default package manager for fedora still version 17 and centos (which is currently 7 and running with yum). From Fedora 18 dnf (Dandified Yum) was introduced . As fedora release by redhat half-yearly the change was made earlier here. And it was a clear indication of that redhat will incorporate the dnf command very soon.

    So what is dnf ?

    The command so called dnf or Dandified Yellowdog updated Modified is the successor of the command yum . DNF's own solution uses SUSE's libsolv , It also uses hawkey as a c and python wrapper.libsolv employees a very solid algorithm for package resolving . This dnf command is also capable of effective represntation of the packaging data in memory . It uses automatic synchronisation of metadata with repositories

    ★hawkeyIt is a package managment library built on the top of libsolv , It is a library allowing clients to query and resolve dependencies of RPM pacjages based on the current state of redhathat package manager database and yum repositories.

    Sometimes it is a ovious question is that Why RPM?because with .tar file any one can install a packages easily . In answer yes it is true any package can be installed , but the package managment process will be slightly difficult . While installing from a .tar file first you have to untar it for this one must tp have good control on the tar command , after unpacking it we have to reslove it dependencies , some times which is really difficult [at last i found it difficult to resplve dependencies manually] . If you are able to fullfil all the installation requirement then you are ready to install the package . But in case of yum command you have to install one simple command yum install package-name and that's all yum will automatically search and install the dependencies along with the package.And like the installation process the uninstallation process will also easy . And one of the best part is that it will store the package insformation

    So The Advantages of RPM installation over .tar installation are

    • No Manuall dependecy resolvence , yum look take care of every thing
    • Similar to installation , uninstallation process is also easy.
    • Installed package database can be viewed , example rpm -qi httpd.

    ★Configuration file for yum repo  /etc/yum/yum.repos.d/repofilename.repo

    Some Essential yum commands

    COMMAND DESCRIPTTION
    yum update This will download and install critical and non-critical updates
    yum install httpd This command will install package httpd (which is Apache http server)
    yum groupinstall mariadb This command will installed all packages in mariadb(selected) group
    yum info httpd This command will display the information about the httpd pacakage
    yum groupinfo mariadb This command will displayed the group information.
    yum list available This command will list all availabel packages
    yum list installed This command will list all installed packages
    yum search httpd This command will search for the httpd package
    yum remove httpd This command will uninstall the httpd package
    yum reinstall httpd This command will reinstall the httpd package
    yum localinstall package-name.rpm This command will locally install the rpm package

    Now for dnf just replace the yum part with dnf , it will work

    example:- yum info kernel to convert it in dnf use dnf info kernel

    Now What is apt?

    apt stands for Advance Package Tool provides highlevel commandline interface package managment system for debian distro linux like UBUNTU. This command will take help of the more specified apt tools apt-get.

    What is apt-get?

    It is the package handling utility of APT , i.e. it is command line tool for handleing packages . The apt-get command is similar to the command yum or dnf . In some cases apt-get is bit more efficient .

    ★Some of the most frequently used apt-get commands

    COMMAND DESCRIPTION
    apt-get update This will download and install critical and non-critical updates
    apt-get upgrade This command is useful for major system upgrades
    apt-get install pip This command will download and install a package called pip
    apt-get remove pip This command will remove installed package called pip
    apt-get purge pip This command will uninstall the packages and also the configuration files will be deleted
    apt-get check This command will check for the broken packages

    rpm and dpkg command's for package managment

    Author:Dipayan Dutta | 07-12-2016

    RPM is the rpm (Redhat Package Manger) command [rpm command work on redhat , fedora , centos but in debian you can install and run rpm command ] ,which can be used to build, install, query, verify, update, and erase individual software packages. this is a very useful utility , unlike the yum or dnf command it doesnot read the .repo file , insetead it works on the rpm files directly . This is command is useful for pakcage query , package installation , package uninstallation and also getting the dependcy information of the packages .. RPM file list store in /var/lib/rpm .

    What is a package ?

    A package is a collection of files and meta-data which is use to install or unistall archive files.

    ★Some of the most useful arguments of the rpm command

    ARGUMENTS DESCRIPTION
    -i installation of the package
    -v prints the verberose information
    -h prints the hash marks of the packets
    --nodeps donot do a dependecy checkup
    -e earase i.e. unistall a package
    -q This is the rpm query argument i.e. get the infos
    -U This is the command for upgradeing a installed package
    -V Verify packages
    --reinstall Reinstall the package
    --version Gives the version information

    ★Some Examples of rpm command

    COMMANDS DESCRIPTION
    rpm -ivh 2hash-0.2-1.el6.rf.i686.rpm This command is used for installing 2hash package
    rpm -q 2hash-0.2-1.el6.rf.i686.rpm This command will check if the package is installed or not
    rpm -qpR 2hash-0.2-1.el6.rf.i686.rpm This will give the information about the dependencies for installing the package
    rpm --nodeps -ivh 2hash-0.2-1.el6.rf.i686.rpm This is will install the package with out any dependencies
    rpm -e --allmatches --nodeps 2hash-0.2-1.el6.rf.i686.rpm This command will unistall the package
    rpm -qa Query all installed packages
    rpm -U 2hash-0.2-1.el6.rf.i686.rpm This command will upgrade the installed package

    dpkg , the package managment command for Debian Distros i.e. install , remove , update , manage the packages .The dpkg command works same as the rpm command , unlike the rpm command works on the .rpm files it works on .deb files.

    ★dpkg will keep track of the installed packages in /var/lib/dpkg/available and the dpkg configuration file is present in /etc/dpkg/dpkg.cfg . To check the status of the available configuration files /var/lib/dpkg/status .

    ★Some most useful command arguments

    ARGUMENTS DESCRIPTION
    -i install a .deb file
    -r removes an installed package , donot remove configuration files
    --purge purge an installed or removed package , also remove the configuration files .
    --verify Verify the integrity of the package
    -list list all the installed packages
    -R Recursively install all the files matching *.deb in a directory
    --print-avail Display the detail information on the package name

    Some useful commands for .deb installation

    COMMANDS DESCRIPTION
    dpkg -i package-name.deb command is use to install a package
    dpkg -r package-name.deb command is use to unistall a package
    dpkg --purge pcakage-name.deb command is use to purge remove a file , removes all config files
    dpkg -list list all available packages
    dpkg -R directory-name install all *.deb files present in the diretory
